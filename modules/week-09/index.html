Week 9: Spatial Analysis — turning GIS operations into policy-relevant insight.

Use this module before class to review concepts, vocabulary, and workflows we’ll apply in lab/exercise.

Learning goals

  • Define spatial analysis and recognize a spatial question.
  • Match common GIS tools (clip, dissolve, buffer, intersect, union) to the question they answer.
  • Explain what a spatial join does and when it can fail (and how to fix it).
  • Describe when you’d move beyond basics: network analysis, interpolation, hotspot detection, spatial regression.

Agenda snapshot

Quick vocabulary (first-use expansions)

  • Geoprocessing: GIS operations that manipulate spatial data to create new data or insight.
  • Extract, Transform, Load (ETL): a common data workflow for preparation and integration.
  • And / Or overlay: Intersect (AND), Union (OR).
  • Spatial join: joining attributes based on a spatial relationship (e.g., within, intersects, closest).
Check yourself: is this a spatial question?

Prompt: “Which neighborhoods should receive a new city-owned grocery store under a food equity initiative?”

Why it’s spatial: the answer depends on location and relationships between layers (demand, access, transit, zoning, parcels, stakeholders).
