Projects and project management in GIS

If you’ve ever struggled with data coordination, shifting scope, or “who changed what,” you’ve already met project management—today we formalize it.

Warm-up: what was hardest to coordinate?

  • Data collection / data cleaning
  • Keeping everyone aligned on scope
  • Version control / “who changed what”
  • Stakeholder feedback changes
  • Time management and deadlines

In class: share one example and how you handled it.

What is a project?

A project is a temporary effort that produces a unique deliverable.

Examples of GIS projects

  • Building a site suitability model for a new facility
  • Creating a web map + dashboard for public reporting
  • Designing a parcel data quality assurance workflow for a city or county

What is project management?

Project management is the use of knowledge, skills, tools, and techniques to deliver something of value to people.

In GIS, this protects quality and reduces rework because datasets, stakeholders, and dependencies are complex.

Quick check

Question: Your team updates a weekly dashboard with the same steps every week. Is that a project?

Show answer

Answer: Usually no — that’s ongoing operations. It becomes a project when you’re building, redesigning, or launching something new.
