Frameworks: Waterfall, Agile, Hybrid

Frameworks are ways to organize work. In GIS, a hybrid approach is often best: structured milestones + iterative analysis.

Waterfall (predictive)

  • Best when requirements are stable
  • Clear phases: define → build → test → deliver
  • Risk: late discovery of data problems

Agile (adaptive)

  • Best when requirements evolve
  • Short sprints + frequent demos
  • Risk: scope creep without guardrails

Hybrid (common fit for GIS)

Use structured planning for fixed components (deadlines, compliance, deliverables) and agile iteration for evolving components (analysis, stakeholder feedback, data discovery).
