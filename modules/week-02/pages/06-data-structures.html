Data structures & formats

Representation & data structures

GIS commonly organizes information into thematic layers—each layer represents one “view” of a theme (parcels, roads, zoning, floodplains…).

Bits, bytes, and why storage matters

Digital spatial data are stored as binary numbers: bits (0/1) grouped into bytes (8 bits). Raster storage grows fast.

File structures: Shapefile vs File Geodatabase

Shapefile: a cluster of files that must stay together (geometry, attributes, index, projection…). If you “mess with” one file, you can break the dataset.

Shapefile File Geodatabase (GDB)
Multiple files working together (.shp, .dbf, .shx, .prj…) Single folder-based database with feature classes/tables
Very compatible across software Supports richer rules/domains/topology
Field name limits and other legacy constraints More flexible schema and larger data support
Vector only Can store vector and raster datasets
Practical takeaway: if you’re doing serious editing, enforcing rules, or managing many related layers, a geodatabase is usually the safer choice.
