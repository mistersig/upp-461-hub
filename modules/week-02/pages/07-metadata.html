Metadata & documentation

Documentation and metadata explain what a dataset is, how it was made, and how (or whether) it should be used.

Planning reality: data with missing metadata is a risk—because you can’t judge accuracy, currency, scale, or suitability.

Minimum metadata you should look for

  • Who created it? (agency / author)
  • When was it created/updated?
  • What is the coordinate system?
  • What’s the scale/resolution and known accuracy?
  • Field definitions + codes (domain meanings)
  • Any processing steps (generalization, interpolation, digitizing)

What metadata prevents

  • Using parcels as if they were legal boundaries when they’re “approximate”
  • Overlaying layers that look aligned but were digitized at different scales
  • Comparing “population” fields that actually mean different years
