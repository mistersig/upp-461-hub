Start here

Goal: understand how real-world phenomena become GIS data, and how representation choices affect analysis.

  • Recognize the two core data models: vector and raster.
  • Explain why topology rules exist (and what “faux” errors look like).
  • Match common variables to nominal / ordinal / interval–ratio measurement scales.
  • Know what a shapefile really is (a cluster of files) and why metadata matters.

Estimated time: 25–40 minutes.

What you’ll do

  • Try a vector vs raster picker
  • Spot topology problems (gaps/overlaps/misalignment)
  • Classify variables by measurement scale
  • Estimate raster storage size
  • Build a metadata checklist you can reuse

Rule of thumb

Before you analyze anything, ask: “What is my phenomenon, and what representation choice did I (or the data provider) make?”

In ArcGIS Pro today (fast check)

  1. Add any dataset (vector or raster) to a map.
  2. Find the layer properties: coordinate system, extent, and attribute fields.
  3. Locate the metadata panel (even if it’s sparse).

You don’t need to master every button—just get comfortable locating key “data about the data.”
