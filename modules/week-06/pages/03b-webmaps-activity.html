Optional activity: Web maps & projections

What you’ll learn Web basemaps are usually displayed in Web Mercator (EPSG:3857), a projected coordinate system used for fast tiled maps. In this activity, you’ll click a location, see its latitude/longitude (geographic coordinates), and then see the corresponding Web Mercator meters.

Why this matters

  • Web Mercator makes it easy to tile/zoom maps quickly.
  • But it distorts area (especially near the poles), so it’s not ideal for analysis.
  • A good workflow: use Web Mercator for display, and a local projection for measurement/analysis.

Accessibility note

The interactive map works best with mouse/touch. If you use a keyboard, use the “Enter coordinates” fields below the map to update the marker.

Why Web Mercator?

It’s widely used for web tiles because it keeps angles locally consistent (conformal) and allows a simple global tiling scheme. The tradeoff is that areas inflate dramatically toward the poles, so it’s not a great choice for measuring area.

Reflection

  • What happens to the Web Mercator Y value as you move north?
  • Why might Web Mercator be a poor choice for comparing areas between Alaska and Florida?
  • For a city-scale analysis in Illinois, what kind of projection would you choose instead?
