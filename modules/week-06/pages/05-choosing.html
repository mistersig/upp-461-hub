5) Choosing the right CRS

Step-by-step checklist

  1. What is the map’s purpose? (area, distance, direction, shape, display)
  2. What is the region? (global vs local; near poles vs mid-latitudes)
  3. What software context? (web basemap? engineering drawing? analysis?)
  4. Pick CRS that fits: datum + coordinate system + units
Common workflow
  1. Set the map/data frame CRS to your target CRS
  2. Define CRS correctly for each dataset (if missing/wrong)
  3. Project datasets into the target CRS for analysis

Example scenarios

  • Choropleth comparing county areas → equal-area
  • Local parcel mapping → local/state plane style CRS
  • Web display with basemaps → Web Mercator (display), but use local CRS for analysis

Common mistakes

  • Using degrees for distance/area
  • Mixing datums without transforming
  • Projecting twice (or defining incorrectly)
