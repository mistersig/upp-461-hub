What is geocoding?

Geocoding converts text-based location descriptions (usually postal addresses) into geographic coordinates so they can be mapped and analyzed.

Definition (plain language) A geocoder reads an address, finds the best matching street/place in a reference dataset, and returns an estimated point.

Why geocoding matters for planners

Practical uses

  • Map incident/service calls (311, inspections, permits)
  • Analyze accessibility (distance to clinics, groceries, transit)
  • Target interventions (heat maps, hotspots, equity metrics)
  • Link administrative data to neighborhoods or parcels

Hidden risks

  • False precision: points can be “close enough” but wrong
  • Bias: informal addresses may not match well
  • Privacy: address data can identify people

From a table → a map

GIS can turn non-spatial tables into spatial layers if the table has a geographic attribute:

  • Shared field (e.g., state name, tract ID, FIPS) → Join attributes
  • Coordinates (lat/long or x/y) → Display XY
  • Addresses → Geocode

We focus on the last case: address-based data.

Check yourself: when is geocoding a bad idea?

Good answer: when you need parcel-level precision but only have a street address range reference, or when addresses are incomplete/informal.

If your decision depends on very precise placement, consider parcel data, rooftop geocoding, or field verification.
