Summary

Geocoding turns address text into coordinates — but the output is only as trustworthy as the input and reference data.

Key takeaways

  • Addresses are social infrastructure. They enable service access and institutional recognition — and can exclude people without formal addresses.
  • Geocoding is a pipeline: preprocessing → matching → interpolating.
  • You need two inputs: an address table + a reference database (streets with address ranges).
  • Interpolation is an estimate along a street segment (not necessarily a rooftop location).
  • Validate results using match rate and spot-check positional accuracy.
  • Think about privacy before using online geocoding services with sensitive addresses.

ArcGIS checklist

  1. Inspect your table: one record per row, unique ID, consistent address fields.
  2. Clean + standardize: fix common spelling/type/ZIP issues.
  3. Choose reference data: TIGER for broad coverage; local for better updates/accuracy.
  4. Create a locator: map fields correctly; set match thresholds.
  5. Geocode + review: rematch “unmatched” and “tied” candidates; document decisions.
  6. Report quality: match rate + known limitations for your analysis.
