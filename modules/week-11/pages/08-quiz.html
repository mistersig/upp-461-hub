Quick quiz

Quick quiz: Use this to check your understanding before class.

Nothing is graded here; focus on the explanations.

1) What are the two key inputs needed for address geocoding?
An address table provides the records to locate, and the reference database provides the streets/ranges used to match and estimate coordinates.
Geocoding needs both your input addresses and a reference dataset that contains street names and address ranges (or other matching geometry).
2) In preprocessing, parsing is best described as…
Parsing isolates address parts so the geocoder can compare them to reference fields consistently.
Interpolation happens after matching. Matching chooses candidates. Travel time is a network analysis task, not geocoding.
3) A match score mainly reflects…
Match scores are based on text/attribute similarity between the input address and candidate features in the reference dataset.
Geocoding match scores aren’t about GPS sensors or your dataset size. They’re about how well the input matches reference candidates.
4) In the interpolation step, what is being interpolated?
Address interpolation estimates a point’s position along a street segment based on where the address number falls within the from–to range.
This interpolation is about placing a point, not estimating a physical measurement or modifying polygons.
5) Which match rate is often considered a reasonable rule-of-thumb for a large dataset (before you inspect errors)?
For large, messy administrative datasets, 85–87% can be “good” — but you still must examine unmatched and low-confidence matches.
Lower match rates may indicate address quality issues, reference gaps, or overly strict thresholds. The goal is not perfection, but defensible quality.
6) Which input is most likely to fail rooftop/street address geocoding?
P.O. Boxes are mail drop locations and don’t correspond to a street frontage position in a street centerline reference.
Street addresses can usually be matched to a street segment and interpolated, assuming the reference dataset includes the street and ranges.
7) What is a key concern when using an online geocoding service for sensitive records?
Online services can log, store, or process addresses under their terms. Consider data sensitivity and policy requirements.
The main issue is not screen settings. Also, online services are not guaranteed accurate or appropriate for all use cases.
