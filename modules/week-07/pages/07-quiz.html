Quick quiz

Try these questions. After you grade the quiz, you’ll get feedback for each question:

  • Why a wrong choice is wrong
1) Which step in Extract, Transform, Load (ETL) most directly involves cleaning column types and fixing missing values?
Transform is where you standardize types, fix missing values, clean categories, and otherwise make data analysis-ready.
Extract is about getting data out of a source, and Load is about writing the (already-prepared) data into a destination. “None” is incorrect because cleaning is a core part of Transform.
2) Why is Geographic Identifier (GEOID) often stored as text instead of a number?
Leading zeros matter (especially for state/county/tract codes). Treating GEOID as a number can drop zeros and break joins.
Join performance depends on the software/database (text is not “always faster”), GIS can join numeric fields, and GEOID is designed to be a stable unique identifier at a specific geography level.
3) American Community Survey (ACS) data are best described as…
ACS is a sample survey. It produces estimates for defined time periods and includes Margin of Error (MOE).
The complete 10-year count is the Decennial Census; ACS is public (not a private paid dataset); and boundary files are spatial products (often from TIGER), not the ACS tables themselves.
4) You joined an ACS tract table to block group boundaries and the map looks wrong. What’s the most likely issue?
Tract tables and block group boundaries represent different units. Mixing levels causes mismatched joins or misleading maps.
Theme settings don’t affect joins; ACS variables are commonly mapped (you must interpret uncertainty); and GEOID is a standard join key for Census data.
5) Which is a classic “not atomic” (non-atomic) value problem in a spreadsheet?
Atomic means one value per cell. Multiple values in one cell should be split into separate rows or a related table.
Single values like “42” are atomic; column naming isn’t a normalization violation by itself; and having a primary key is good practice.
6) Topologically Integrated Geographic Encoding and Referencing (TIGER) is best described as…
TIGER products are spatial extracts from the Census Bureau’s database (boundaries, roads, and other features).
TIGER is data (spatial features), not the ACS estimation process, not a cleaning tool, and not the same thing as ACS.
7) In the normalization checklist, which field is most important to identify first?
A primary key anchors the table: each row represents one entity, and other fields should depend on that key.
“Interesting,” “most decimals,” or “many blanks” might be worth investigating later, but normalization starts by defining what a row represents (the key).
8) If you map an ACS estimate, what’s a responsible next step before making strong claims?
ACS estimates have uncertainty. Interpreting MOE and variable definitions (and matching geography) helps avoid overclaiming based on noisy or mismatched data.
Ignoring uncertainty can be misleading; converting everything to text is unnecessary; and deleting nulls without understanding them can bias results.
