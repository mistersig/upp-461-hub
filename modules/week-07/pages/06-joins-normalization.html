Joins + normalization

Join by attribute (Join by Attribute)

To map Census tables, you typically join a CSV to a boundary layer using a shared key (often GEOID).

  • One side must have a unique identifier (no duplicates).
  • The key must match exactly (types, leading zeros, formatting).

Common join failure modes

  • Leading zeros dropped (treating IDs as numbers).
  • Duplicates in the join key (causes one-to-many surprises).
  • Different geography levels (tract table joined to block group boundaries).
  • Stale boundaries (different vintage/year).

Codd-style normalization checklist (adapted for GIS data)

Check yourself: You see “Chicago, IL; Evanston, IL” in one cell. What’s the normalization problem?

Answer: The cell is not atomic (it contains multiple values). Split into separate rows or a related table.
