Interoperability & formats

Standards: Open Geospatial Consortium (OGC)

The Open Geospatial Consortium (OGC) develops standards so different GIS tools and services can exchange data reliably (think: consistent ways to describe geometry, services, and common formats).

Common GIS file types (and what to watch)

Comma-Separated Values (CSV)
Great for tables. Watch: types, delimiters, encoding, and join keys.
Shapefile
Very common legacy vector format. Watch: multiple companion files (.shp/.shx/.dbf/.prj), field name limits, and encoding.
GeoPackage
A modern single-file container (SQLite). Often better than Shapefile for projects.
GeoJSON
Common for web mapping. Watch: size and precision; not ideal for very large datasets.
Check yourself: Why can a Shapefile “break” if you only move the .shp?

Answer: A Shapefile is a set of multiple files. If you don’t move the companion files (especially .dbf, .shx, and .prj), attributes and spatial indexing/projection info can be lost.
