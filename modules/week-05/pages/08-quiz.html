Quick quiz

Quick quiz: Check your understanding before class.

Nothing is graded here; focus on the explanations.

1) In an attribute table, a row usually represents…
A row (record) contains all attributes for one feature.
Fields are columns; projections and spatial relationships are not rows.
2) Which where clause correctly selects Cook County when COUNTYFP is text?
Text values usually need quotes, and leading zeros matter for codes like FIPS.
Without quotes you may lose leading zeros or get type mismatch.
3) Using AND instead of OR usually makes a selection…
AND requires both conditions to be true, reducing matches.
OR matches either condition, often increasing matches.
4) Which selection type keeps the current selection and adds new matches?
“Add to selection” unions new matches with existing selection.
New replaces; remove subtracts; subset narrows.
5) “Within a distance of” is conceptually like…
It’s a proximity rule—like selecting features inside a buffer around the source.
Joins/sorting/projection are not spatial proximity rules.
6) For a join to work reliably, join fields must…
Type + formatting (text vs number, zeros, spaces) affect matching.
Names and ordering do not guarantee key matching.
7) After a join, most joined fields are null. Best first move?
Nulls usually mean the key values didn’t match due to formatting/type issues.
Basemaps/projection/raster conversion don’t fix key mismatches.
